## 1. 为什么要学习建造者模式？
有时候，一个高度复合的对象有几个拥有多种变化可能的内部抽象成员构成。      
其中抽象成员的构建方式和算法可能多变且复杂，但将这几个抽象成员的组合关系却相对简单且保持恒定。     
这时候，将对象内部成员的复杂构建情形抽离出来，将高度复合对象的构建过程进行抽象化处理，对调用者屏蔽底层细节，有利于获取多变组合的对象的实例。
